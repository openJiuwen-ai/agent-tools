# OpenAPI （插件市场）

## 接口规范文档

```yaml
openapi: 3.1.0
info:
  title: 插件市场 API
  description: 插件上传、发布、查询、删除与下载接口；含发布页模板 zip 的预签名下载（私有桶对象）
  version: 1.0.0
servers:
  - url: http://localhost:8100
    description: 本地开发环境
  - url: https://market.example.com
    description: 生产环境
components:
  securitySchemes:
    bearerAuth:
      type: http
      scheme: bearer
    systemTokenAuth:
      type: apiKey
      in: header
      name: X-System-Token
paths:
  /api/v1/plugins:
    post:
      summary: 发布插件
      description: 发布插件到指定空间，同时完成文件上传和发布操作。鉴权需二选一：Authorization Bearer 或 X-System-Token（必须且只能提供一个）。
      operationId: publishPlugin
      tags:
        - 插件管理
      security:
        - bearerAuth: []
        - systemTokenAuth: []
      parameters:
        - name: X-Checksum-SHA256
          in: header
          required: true
          description: 插件包文件的 SHA256 校验和（小写十六进制字符串）
          schema:
            type: string
            pattern: "^[a-f0-9]{64}$"
            example: "a1b2c3d4e5f67890a1b2c3d4e5f67890a1b2c3d4e5f67890a1b2c3d4e5f67890"
        - name: Authorization
          in: header
          required: false
          description: Bearer Token（与 X-System-Token 必须且只能提供一个）
          schema:
            type: string
            example: "Bearer {permanent_token}"
        - name: X-System-Token
          in: header
          required: false
          description: 系统 Token（与 Authorization 必须且只能提供一个）
          schema:
            type: string
      requestBody:
        required: true
        content:
          multipart/form-data:
            schema:
              type: object
              required:
                - file
              properties:
                file:
                  type: string

          description: 发布成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/PluginPublishResponse'

                message: "Publish plugin successfully"

                  name: "weather-plugin"
                  version: "1.0.0"
                  status: "ACTIVE"
                  published_at: "2025-01-01T00:00:00Z"

                    code: 400
                    data: null
                    error: "checksum_mismatch"
                    message: "文件校验和不匹配，文件可能在传输过程中损坏"
        '401':
          description: 未授权 / token 无效
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
        '403':
          description: 权限不足
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
              example:
                code: 403
                data: null
                error: "permission_denied"
                message: "您没有权限在该空间发布插件"
        '404':
          description: 插件不存在
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
              example:
                code: 404
                data: null
                error: "plugin_not_found"
                message: "插件 '3589119244ed45c29f98038642872858' 不存在，无法添加新版本"
        '409':
          description: 版本冲突
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/VersionConflictResponse'
              examples:
                version_conflict:
                  summary: 版本冲突（应用层检测）

                    code: 409

                        plugin_id: "3589119244ed45c29f98038642872858"
                        version: "1.0.0"
                    error: "version_conflict"
                    message: "插件 'weather-plugin' 版本 '1.0.0' 已存在，如需覆盖请设置 force=true"
                version_exists:
                  summary: 版本已存在（数据库约束触发）
                  value:
                    code: 409
                    data:
                      existing_version: "1.0.0"
                    error: "version_exists"
                    message: "插件版本 '1.0.0' 已存在，如需覆盖请设置 force=true"
                plugin_name_exists:
                  summary: 插件名称已存在
                  value:
                    code: 409
                    data: null
                    error: "plugin_name_exists"
                    message: "您已发布过同名插件 'weather-plugin'，请使用其他名称或为现有插件添加新版本"
        '413':
          description: 文件过大
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
              example:
                code: 413
                data: null
                error: "file_too_large"
                message: "文件大小超过限制（最大512MB）"
        '422':
          description: 请求验证失败
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
              examples:
                manifest_validation_failed:
                  summary: 版本号格式错误
                  value:
                    code: 422
                    data: null
                    error: "manifest_validation_failed"
                    message: "版本号格式错误，必须为 <主版本号>.<次版本号>.<修订号>，例如 1.0.0、1.0.1（不应有 v 前缀）"
                plugin_id_mismatch:
                  summary: plugin_id 与插件包不一致（填写的 plugin_id 与 name 对应的插件不匹配）
                  value:
                    code: 422
                    data:
                      expected_plugin_id: "plugin_xyz789"
                    error: "plugin_id_mismatch"
                    message: "plugin_id 与插件包不匹配：您填写的 plugin_id='3589119244ed45c29f98038642872858' 与插件名称 'weather-plugin' 对应的插件不一致"
                plugin_id_required:
                  summary: 发布新版本时必须填写 plugin_id
                  value:
                    code: 422
                    data:
                      expected_plugin_id: "3589119244ed45c29f98038642872858"
                    error: "plugin_id_required"
                    message: "当前名称下已存在插件，发布新版本时必须填写 plugin_id"
        '500':
          description: 服务器内部错误
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
              examples:
                internal_server_error:
                  summary: 服务器内部错误
                  value:
                    code: 500
                    data: null
                    error: "internal_server_error"
                    message: "服务器内部错误，请稍后重试"
                storage_error:
                  summary: 存储上传失败
                  value:
                    code: 500
                    data: null
                    error: "storage_error"
                    message: "插件包上传失败"
    get:
      summary: 获取插件列表

        支持分页、筛选与排序（支持 `plugin_type` 精确匹配与 `search_keyword` 模糊搜索等）。

        - 插件管理

          required: false

          schema:
            type: integer
            minimum: 1
            maximum: 100
            example: 20
        - name: asset_id
          in: query
          required: false
          description: 资产 ID
          schema:
            type: string
        - name: asset_type
          in: query
          required: false
          description: 资产类型
          schema:
            type: string
        - name: publisher_id
          in: query
          required: false
          description: 发布者 ID
          schema:
            type: string
        - name: publisher_name
          in: query
          required: false
          description: 发布者名称（模糊）
          schema:
            type: string
        - name: plugin_type
          in: query
          required: false
          description: 插件类型（精确匹配）
          schema:
            type: string

                  code:

                    example: 200
                  message:

                    example: ok

          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
        '500':
          description: 服务器内部错误

  /api/v1/plugins/publish-template:
    get:
      summary: 获取发布页插件模板 zip 预签名下载链接
      description: |
        为前端「发布插件」页提供模板包下载：服务端根据配置的桶内对象 Key（环境变量 `MARKET_PLUGIN_TEMPLATE_OBJECT_KEY`）生成 **预签名 GET URL**，对象可放在 **私有** MinIO/OBS 桶中。

        可选：`MARKET_PLUGIN_TEMPLATE_PRESIGNED_EXPIRES`（秒）单独控制本链接 TTL；为 0 或未设置时沿用 `MARKET_S3_PRESIGNED_EXPIRES`。
        客户端应在用户点击下载时再请求本接口，避免预签名过早过期。
      operationId: getPublishTemplatePresigned
      tags:
        - 插件管理
      security:
        - bearerAuth: []
        - systemTokenAuth: []
      parameters:
        - name: Authorization
          in: header
          required: false
          description: Bearer Token（与 X-System-Token 必须且只能提供一个）
          schema:
            type: string
            example: "Bearer {permanent_token}"
        - name: X-System-Token
          in: header
          required: false
          description: 系统 Token（与 Authorization 必须且只能提供一个）
          schema:
            type: string
      responses:
        '200':
          description: 成功返回预签名 URL 及建议文件名
          content:
            application/json:
              schema:
                type: object
                required: [code, message, data]
                properties:
                  code:
                    type: integer

                  message:
                    type: string
                    example: ok
                  data:

              example:
```
